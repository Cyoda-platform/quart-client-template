### Product Requirements Document (PRD) for Cyoda Design

#### Introduction

This document outlines the requirements for the Cyoda application, specifically focusing on the design that enables the aggregation of BTC/USDT rates from the Binance API. The goal is to create a system that efficiently retrieves, stores, and processes cryptocurrency exchange rates, allowing users to perform aggregations and view the data as needed.

#### User Requirement

**Requirement Statement**: Create an application that retrieves the BTC/USDT exchange rate from the Binance API every 5 seconds, stores the data in a database, and allows for future aggregations and chart displays based on user requests.

### User Stories

1. **As a User**, I want to receive real-time updates of the BTC/USDT exchange rate from the Binance API every 5 seconds so that I can monitor price fluctuations.
2. **As a User**, I want to store the retrieved exchange rates in a database so that I can access historical data for analysis.
3. **As a User**, I want to trigger an aggregation function that calculates the average exchange rate, so I can make informed decisions based on the data.
4. **As a User**, I want to view the aggregated data in charts over different timeframes (5 min, 30 min, 1 day, 1 week) for better visualization of trends.

### Entities Overview

1. **Data Ingestion Job**
   - **Description**: Responsible for fetching the BTC/USDT rate from the Binance API.
   - **Saving Method**: Triggered by a scheduled job or an API call.

2. **Raw Data Entity**
   - **Description**: Stores the unprocessed data obtained from the Binance API.
   - **Saving Method**: Saved via an ENTITY_EVENT generated by the Data Ingestion Job.

3. **Aggregated Data Entity**
   - **Description**: Contains the average values calculated from the Raw Data.
   - **Saving Method**: Generated through a workflow associated with a user action.
